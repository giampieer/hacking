Metodologías de pentesting

El Pentesting puede ser o no ético dependiendo su fin, si es para el bien o para el mal. La metodología de un Pentest se divide entre: WhiteBox, BlackBox y GrayBox. White Box Testing La auditoría de WhiteBox (caja blanca) se realizan con el fin de simular un ataque a la infraestructura de una empresa. En este tipo de pruebas el Pentester conoce el funcionamiento del sistema internamente, así mismo trabaja con información que puede tener de uno o varios empleados. La auditoría de BlackBox (caja negra) se realizan sin tener ningún tipo de información con respecto de la empresa y es el Pentester quien debe descubrir las vulnerabilidades en la infraestructura. La auditoría de GrayBox (caja gris) se realizan con algunos conocimientos del funcionamiento de un sistema informático o no, los Pentesters en este caso mezclan las características de los dos métodos anteriores. En Ciberseguridad un vector de ataque es una ruta o camino que utiliza un atacante para obtener acceso al sistema informático, en otras palabras, son los pasos a seguir del Pentester para materializar su ataque. APT (Amenaza avanzada persistente): Este tipo de ataques es uno de mayor gravedad a los habituales porque existe un atacante (o un grupo) que tiene un objetivo ilícito muy bien establecido, las APT poseen características que hacen que sus efectos dañinos sean más fuertes. El atacante que usa este tipo de vector de ataque es muy paciente (y con conocimientos avanzados de programación y seguridad informática) ya que se da en un periodo de tiempo largo, este tipo de ataques puede durar incluso años. Delincuentes informáticos Botnet: En este tipo de ataque se usan virus troyanos especiales para violar la seguridad de las computadoras de varios usuarios, así de esta forma tomar el control de cada una de ellas y organizarlas en una red de manera que el atacante pueda administrarlas de forma remota. La palabra Botnet se forma a partir de las palabras "robot" y "red". Ataques internos: Este tipo de ataque son aproximadamente el 80% de los ataques informáticos que se realizan, es decir, son los más comunes y son los que más impacto pueden generar en la infraestructura de alguna empresa ya que por ser un ataque internos se es más fácil acceder a los sistemas informáticos. Amenazas Móviles: Este tipo de ataque parte de la tendencia de llevar consigo un dispositivo móvil y por medio de ellos acceder a la información, a la red de una empresa.

Algunos términos importantes que se deben conocer son: ¿Qué es un activo?: Es todo lo valioso para una empresa. ¿Qué es una amenaza?: Es una falla que puede ocurrir en una empresa, incluso fallas que existen pero no se conocen. ¿Qué es una vulnerabilidad?: Es una debilidad de un sistema informático que puede ser utilizada para causar daños. ¿Qué es un riesgo?: Es un proceso que comprende la identificación de las vulnerabilidades y amenazas dando lugar a un ataque. ¿Qué es un exploit?: Es el proceso de tomar las fallas que existe en el sistema y sacarle provecho. Que es el Payload?:Es la ejecución que se realiza dentro del sistema una vez aplicado el exploit.

Las personas que trabajan en el tema de seguridad informática normalmente se les conoce como Hackers pero la mayoría del tiempo se piensa en ellos como delincuentes informáticos, sin embargo existen tres tipos de Hackers: Hacker de sombrero blanco (White Hat Hacker): Estos están autorizados para acceder a un sistema, para realizarle pruebas de seguridad, entre otras cosas que garanticen la seguridad de los sistemas de una empresa. Hacker de sombrero negro (Black Hat Hacker): Estos no están autorizados para acceder a un sistema, al no tener permisos de acceso están cometiendo un delito. Hacker de sombrero gris (Gray Hat Hacker): Estos están entre el bien y el mal, es decir, pueden tener acceso a un sistema y cometer acciones ilícitas con el fin de obtener algún beneficio.
